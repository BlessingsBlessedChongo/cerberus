================================================================================
API ENDPOINTS
================================================================================

Authentication
├─ POST /api/auth/login/
│  ├─ Request: { username: string, password: string }
│  ├─ Response: { token: string, username?: string, user_id?: number }
│  └─ Header: Content-Type: application/json

Metrics
├─ GET /api/metrics/latest/?page_size=5
│  ├─ Response: { results: [{metric_type, value, timestamp}, ...] }
│  ├─ Metrics: CPU_Usage, Available_RAM, Disk_Free_C
│  └─ Header: Authorization: Token {token}
│
├─ GET /api/metrics/history/?hours=24&type=CPU_Usage
│  ├─ Response: { results: [{value, timestamp}, ...] }
│  ├─ Order: Descending (newest first)
│  ├─ Format: Results MUST be reversed in React
│  └─ Header: Authorization: Token {token}
│
└─ GET /api/metrics/history/?hours=24&type=Available_RAM
   ├─ Response: { results: [{value, timestamp}, ...] }
   ├─ Order: Descending (newest first)
   └─ Header: Authorization: Token {token}

Alerts
└─ GET /api/alerts/
   ├─ Response: { results: [{id, timestamp, text_value/value}, ...] }
   └─ Header: Authorization: Token {token}

================================================================================
DATA FLOW
================================================================================

LOGIN PAGE:
1. Component mounts
2. Boot sequence begins (useEffect)
3. Terminal lines display character-by-character (50ms/char)
4. After line completes → add to terminalLines array
5. 500ms delay after ANALYTICS line
6. Final line completes → showForm state = true
7. Form fades in (CSS opacity transition 0.7s)
8. User enters credentials
9. Submit → POST /api/auth/login/
10. Success → save token to localStorage
11. Navigate to dashboard (/)

DASHBOARD PAGE:
1. Component mounts
2. Check token from AuthContext
3. If missing → redirect to /login
4. setInterval for metrics (10 second polling)
5. Fetch /metrics/latest/ → parse and display
6. Fetch /metrics/history/ → reverse → chart
7. Fetch /alerts/ → display list
8. Every 10 seconds → loadLatest() fires
9. Charts update automatically
10. Metrics display updates
11. On logout → clear localStorage → navigate to /login

================================================================================
ERROR HANDLING FLOWS
================================================================================

Network Error Flow:
1. Fetch fails (network unavailable)
2. Catch block triggered
3. handleFetchError() called
4. isConnected set to false
5. Connection notice displayed
6. Metrics show "--"
7. Charts show "No data available"
8. User sees helpful message
9. No page crash

Login Error Flow:
1. Form submitted
2. isLoading set to true (button disabled)
3. Fetch POST /api/auth/login/
4. Response not OK
5. Extract error from response.detail
6. setError() with message
7. Catch block catches additional errors
8. isLoading set to false (button re-enabled)
9. User sees red error alert
10. Can retry immediately

Chart Data Error Flow:
1. Fetch /metrics/history/ fails
2. Catch block triggered
3. cpuHistory remains null
4. Chart displays "Loading..." → "No data available"
5. Dashboard remains functional
6. Next 10-second poll will retry
