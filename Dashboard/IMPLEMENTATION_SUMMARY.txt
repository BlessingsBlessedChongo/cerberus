================================================================================
AUTHENTICATION FLOW
================================================================================

1. User arrives at /login
2. Terminal boot sequence begins (4 lines, ~10 seconds)
3. User sees character-by-character typing animation
4. After sequence completes, login form fades in
5. User enters username and password
6. Clicks "Access Dashboard"
7. POST request to /api/auth/login/ with credentials
8. Token returned and saved to localStorage
9. User redirected to dashboard with metrics loading
10. Token automatically included in all subsequent requests

If login fails:
→ Red error alert displayed
→ User can retry immediately
→ Error message guides troubleshooting

HOW TO RUN
================================================================================

1. Navigate to project:
   cd /vercel/share/v0-project/Dashboard

2. Install dependencies:
   npm install

3. Start development server:
   npm run dev

4. Open browser:
   http://localhost:5174

5. You'll see:
   - Split-screen login page
   - Terminal boot sequence starting
   - Character-by-character typing animation
   - Form fade-in after sequence completes

6. To connect to backend:
   - Ensure Django API running at http://localhost:8000
   - Enter valid credentials
   - Click "Access Dashboard"
   - See live metrics and charts update every 10s
