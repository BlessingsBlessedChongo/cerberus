┌──────────────────────────────────────────────────────────────────────────────┐
│                    INTERACTION FLOWS                                         │
└──────────────────────────────────────────────────────────────────────────────┘

LOGIN FLOW:
1. User lands on /login
2. Terminal begins typing animation (400ms per line)
3. User enters username & password
4. User clicks "Access Dashboard" button
5. Form disables, spinner shows
6. Request sent to POST /api/auth/login/
   ├─ Success: Token saved, navigate to /
   └─ Failure: Error alert displays, form re-enables
7. On /, user sees dashboard

DASHBOARD FLOW:
1. User logged in, sees dashboard
2. On mount:
   ├─ Load latest metrics
   ├─ Load CPU history (24h)
   ├─ Load RAM history (24h)
   └─ Load alerts
3. Chart renders with data
4. Every 10 seconds: Latest metrics update
5. Connection status updates based on fetch success
6. If disconnected: Notice banner shows
7. User clicks "Logout":
   ├─ Clear token & user from localStorage
   ├─ Clear context state
   └─ Navigate to /login

CARD INTERACTIONS:
- Hover: Border color changes to cyan, shadow increases
- Metrics update: Smooth number transition
- Loading: Shows "--" while fetching

CHART INTERACTIONS:
- Hover: Tooltip shows exact values
- Points: Highlight on hover
- Legend: Click to toggle series

ERROR HANDLING:
- Network error → "Connection notice" banner
- Invalid login → Red alert with error message
- Missing data → Shows "--" or placeholder
